# Contributing to Axiom MLOps Projects

Thank you for contributing to the VodaPay Funding Prediction Engine!

## Technical Standards
- **Environment:** Use the provided `.devcontainer` for all development.
- **Dependency Management:** Use `uv` for adding any new packages.
- **Formatting:** All code must pass `ruff` formatting and `bandit` security checks.

## Pull Request Process
1. Create a descriptive feature branch (`feature/your-feature`).
2. Ensure all `pre-commit` hooks pass locally.
3. Open a Pull Request with a clear description of the "What" and "Why".
4. Address any feedback from the automated Gemini Code Assist or human reviewers.
5. Once approved, the PR will be merged into `main`.

## "Good First Issues"
Look for issues with the `good-first-issue` label to get started with the codebase!
